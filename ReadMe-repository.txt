
# Repository structure:

1/ dist: contain the zip distribution file (i.e. binary version of the software)

2/ docs: documentation (e.g. technical report, manual, other document resources etc.)

3/ lib: library files needed to run src/ (mostly .jar files)

4/ src: source code (e.g. model package of the domain model)

5/ src.example (optional): example source code 

6/ test: test code of the software
